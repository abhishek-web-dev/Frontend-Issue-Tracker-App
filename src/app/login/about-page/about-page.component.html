Problem Statement

This is Issue Tracking Web Application. Which provide a platform, where user can post a query/issue/bug and communicate with other users.

Proposed system

This web application has these many features:

Login view

  1. User will be able to login to the application through his email and password.
  2. User will be able to create a new account on this platform.
  3. Once user will login, redirect to his personalized dashboard view.

Personalized Dashboard view

A table showing all issues currently available on this platform. User can filter the issue using filter option is given. User can see how many issues are available in which category. Like In-Backlog, In-Progress, In-Test, Done. This table has following columns -

  1. Title: Title of the issue.
  2. Status: Current status of the issue. It can be In-Backlog, In-Progress, In-Test or Done.
  3. Reporter: User who has reported the bug/issue.
  4. Date: Date when this issue/bug was reported.
  5. A Search Box: Where, user will be able to search an issue.
  6. A Create Button: User will be able to create a new issue/bug.
  7. Select Option: In this column user can select either View/Edit option. This option will allow to view or edit any query.

Issue description view

  1. Here user (Reporter) will be able to add/edit title of issue, description of issue, add/edit any related attachments, like screenshots.
  2. Description box have a rich text editor having option like font styling, underline etc.
  3. Any user can add any issue to his watch list after that he will get notification, if any other user will change or comment on that issue.
  4. An assignee/reporter can assign this issue to any other user.
  5. Only reporter can delete the issue, which one he has created.
  6. Only reporter/assignee can edit an issue, which has created by them or assign to them.
  7. Any user can view an issue.
  8. There is a comment ysection, where any user, will be able to make a comment on any issue.
  9. User can see all previous comments done by other users.

Search view

  1. User will be able to search for any text.
  2. User will be able to see result table, having all the issues related to the search text.
  3. User can sort any column data
  4. User can also filter table issue on the basis of -
    1. Issue/Bug created by user
    2. Issue/Bug assign to user
    3. All issue/bug

Notification

  1. User can see all notification on one click.
  2. Once user will click any notification, user will redirect to edit page.
  3. Once user will click notification, that notification seen button become blue otherwise that will be light gray.
  4. User will have one option to see all notification. Where user can see all notification in details.

User Watch list

  1. User can see all the issue which one he has subscribed.
  2. User can also View/Delete his watch list query.
